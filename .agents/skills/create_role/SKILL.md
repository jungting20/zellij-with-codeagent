# 역할 자동 생성 스킬 (Create Role Skill)

본 문서는 `zellij-with-codeagent` 프로젝트에서 새로운 에이전트 역할을 자동 생성하고 시스템에 등록하는 스킬 명세서입니다. 에이전트는 이 문서에 명시된 절차와 규칙에 따라 사용자와 질답을 진행하고 코드를 자동으로 작성 및 검증해야 합니다.

---

## 1. 사용자 질답 (Q&A) 단계
에이전트는 정호님께 순차적으로 질문을 던져 새로운 역할에 대한 요구사항을 명확히 정의합니다. 아래 정보를 수집해야 합니다:
1. **역할 이름 (Role Name)**: 영문 소문자와 하이픈(-) 조합 (예: `logger`, `reporter`, `resource-monitor`)
2. **상세 기능 설명**: 이 역할이 담당할 핵심 기능 요약
3. **Chromedp 기반 데이터 수집 여부**: 실제 웹 페이지 브라우징 및 데이터 수집(`chromedp` 라이브러리 사용)이 필요한지 여부 (Yes/No)
4. **타겟 URL (Target URL)**: chromedp 사용 시 분석하거나 트래킹할 기본 URL 주소
5. **UI 및 로그 스트리밍 스타일**:
   - **Option A (대시보드형)**: 주기적(예: 800ms)으로 화면을 지우고(`ui.ClearScreen()`), 진행 바와 통계 로그를 출력하는 대시보드 레이아웃.
   - **Option B (스트리밍형)**: 이벤트 발생 시마다 로그를 한 줄씩 스트리밍하여 출력하는 타임라인 레이아웃.

---

## 2. 코드 자동 생성 가이드
수집된 명세를 바탕으로 Go 소스 코드를 자동 생성합니다. 코드를 생성할 때는 반드시 다음 규칙들을 준수해야 합니다:

### A. 디렉토리 구조 및 패키지 정의
- **생성 경로**: `cmd/agent-role/{role_name}/{role_name}.go` (하이픈은 Go 패키지 규칙에 맞추어 언더바 `_`로 치환하거나 제거하여 단일 소문자 단어로 패키지명 작성)
- **패키지 선언**: `package {role_package_name}`
- **임포트 구성**: `ui` 공통 모듈(`zellij-with-codeagent/cmd/agent-role/ui`)을 반드시 임포트하고, chromedp 사용 시 관련 패키지를 포함시킵니다.

### B. UI 디자인 규격 준수 (중요)
- 화면 지우기 동작은 `ui.ClearScreen()`을 호출합니다.
- CLI 텍스트 출력 시 `ui.Blue`, `ui.Green`, `ui.Yellow`, `ui.Red`, `ui.Reset` 등의 ANSI 색상 상수를 조합하여 시각적으로 우수하고 정돈된 대시보드를 제공해야 합니다.
- 게이지바나 로딩률 표시는 `ui.GenerateProgressBar(percent, width)` 함수를 활용합니다.

### C. 에러 처리 및 안정성 확보 (필수)
- **Null 포인터 방지**: Go 포인터 변수(예: flag로 전달받은 포인터, 구조체 참조 등)에 접근할 때는 **반드시 사전에 `nil` 체크**를 수행해야 합니다.
- **예외 복구**: 에러 발생 시 프로그램이 완전히 비정상 종료(Panic)하지 않도록 복구(recover) 메커니즘을 두거나, 내부 루프 내에서 에러 발생 시 로그만 출력하고 `continue` 하도록 설계합니다. 심각한 초기화 실패의 경우에만 `os.Exit(1)`을 적용합니다.
- **함수형 패턴**: 부수 효과가 없는 순수 함수 구조를 지향하며, 복잡한 비즈니스 로직은 `Run` 함수 내부가 아닌 외부 헬퍼 함수로 격리하여 작성합니다.

---

## 3. 메인 라우팅 자동 등록 가이드
작성된 패키지를 실행 파일에 매핑하기 위해 `cmd/agent-role/main.go` 파일을 수정합니다. 전체 파일을 덮어쓰지 않고, 필요한 부분만 정밀하게 치환(diff 기반)해야 합니다:
1. **임포트 블록 추가**: 상단 import 그룹에 `"zellij-with-codeagent/cmd/agent-role/{role_name}"` 패키지를 추가합니다.
2. **도움말 추가**: `printUsage()` 함수 내에 새로운 역할 명령어 포맷과 설명을 추가합니다.
3. **Switch Case 매핑**: `main()` 내 `switch role` 블록에 신규 case 문을 매핑합니다.
   - 인자(플래그) 파싱이 필요하다면 `run{RoleName}(os.Args[2:])` 형태의 별도 파라미터 처리 함수를 `main.go` 하단에 추가하여 처리합니다. 파싱 시에도 포인터 변수의 nil 체크를 잊지 마십시오.

---

## 4. 빌드 검증 및 장애 대응 (Compile & Fallback)
1. **자동 빌드 수행**: 코드 작성이 끝나면 터미널에서 아래 명령어를 실행하여 컴파일 여부를 확인합니다.
   ```bash
   go build -o bin/agent-role ./cmd/agent-role
   ```
2. **빌드 오류 대응**: 빌드 실패 시 출력되는 오류 메시지를 연쇄적으로 분석하여 해당 소스 코드 파일의 에러 라인만 찾아내어 수정(diff 기반 수정)합니다.
3. **자동 롤백**: 수정 시도 이후에도 지속적으로 컴파일이 실패할 경우, 변경 사항을 `git checkout` 혹은 백업본을 통해 안전하게 원상태로 롤백하고 작업 결과를 사용자에게 리포트합니다.
